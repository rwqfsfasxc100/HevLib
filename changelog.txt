[1.15.14]

1=* Added `EVENT_DRIVER.gd` driver.
1.1=* Each constant lets you add an event from any Vanilla event scripts, through a custom event provided, or lets you change the event timer.
1.2=* For events, must have a `event_name` and `event_type` properties, and all events can use an optional `custom_property_modifications` dictionary.
1.2.1=* `event_name` is the name for the event, and since is used for the node name for the event, can conflict with other events of the same name. Ensure you use a unique event name.
1.2.2=* `event_type` is used for either the vanilla event type, or `script` to provide the event's script path through a `script_path` property.
1.2.3=* `custom_property_modifications` can be used by all events, and is structured with the literal property names as the keys and the set properties as the values.
1.2.3.1=* Useful for setting properties not exposed by this driver, modifying properties added by modifying vanilla events, or is required for changing properties of `script` event types.
1.3=* For changing the event timer, used only the `event_type` property set to `event_delay`
1.3.1=* `delay_time` property is the time that this modification uses
1.3.2=* `operation` is the method used to modify. If not a valid defined operation, fails and does not get changed. Defaults to `set`.
1.3.2.1=* `set` - Overrides the current timer as defined with `delay_time`.
1.3.2.2=* `add` - Adds the time defined with `delay_time`.
1.3.2.3=* `subtract` - Removes the time defined with `delay_time`.
1.3.2.4=* `multiply` - Multiplies the current timer time by the value defined with `delay_time`.
1.3.2.5=* `divide` - Divides the current timer time by the value defined with `delay_time`.
1.4=* Since there are over 20 unique vanilla events, and many have a large number of properties, the individual properties won't be explained here.
1.4.1=* A script contains examples and documentation for all events, including examples of other parts.
1.4.2=* `res://HevLib/events/custom_events/event_defaults.gd`
2=* Fixed title screen changelog menu having oversized spacing between labels
3=* Fixed `DataFormat`.`__split_array_by_length` outputting junk data on arrays smaller than the splitting size.
4=* `NodeAccess`.`__exit` can now have it's exit/restart operation delayed with a `delay` float for the delay in seconds
